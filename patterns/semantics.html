Semantic Structure Patterns

Examples demonstrating semantic HTML5 elements for better document structure and accessibility.

What is Semantic HTML?

Semantic HTML uses elements that clearly describe their meaning to both the browser and the developer. Instead of using generic div and span elements everywhere, semantic elements like header, nav, main, article, section, aside, and footer provide context about the content they contain.

Benefits of semantic HTML include:

  • Improved accessibility for screen readers
  • Better SEO (search engines understand content structure)
  • Easier code maintenance and readability
  • Clearer document outline

Header Element

The header element represents introductory content, typically containing heading elements, logos, navigation, or search forms.

Example Header Structure

<header>
    <h1>Website Title</h1>
    <p>Tagline or description</p>
    <nav>
        <ul>
            <li><a href="#">Home</a></li>
            <li><a href="#">About</a></li>
            <li><a href="#">Contact</a></li>
        </ul>
    </nav>
</header>

Nav Element

Example Navigation

<nav>
    <ul>
        <li><a href="index.html">Home</a></li>
        <li><a href="about.html">About</a></li>
        <li><a href="services.html">Services</a></li>
        <li><a href="contact.html">Contact</a></li>
    </ul>
</nav>

Main Element

The main element represents the dominant content of the page. There should be only one main element per page, and it should not be nested inside header, footer, nav, aside, or article elements.

Example Main Structure

<main>
    <h1>Page Title</h1>
    <p>This is the main content of the page.</p>
    
    <section>
        <h2>Section Heading</h2>
        <p>Section content goes here.</p>
    </section>
</main>

Article and Section Elements

The article element represents a self-contained composition that could be independently distributed or reused (e.g., blog post, news article). The section element represents a thematic grouping of content, typically with a heading.

Article Example

<article>
    <h2>Blog Post Title</h2>
    <p>Published on <time datetime="2026-01-20">January 20, 2026</time></p>
    <p>This is the content of the blog post...</p>
    
    <section>
        <h3>Comments</h3>
        <p>Comment content...</p>
    </section>
</article>

Section Example

<section>
    <h2>Chapter 1: Introduction</h2>
    <p>Content of chapter 1...</p>
</section>

<section>
    <h2>Chapter 2: Getting Started</h2>
    <p>Content of chapter 2...</p>
</section>

Aside Element

The aside element represents content that is tangentially related to the main content, such as sidebars, pull quotes, or advertisements.

Example Aside

<article>
    <h2>Main Article Content</h2>
    <p>This is the main article text...</p>
    
    <aside>
        <h3>Related Links</h3>
        <ul>
            <li><a href="#">Related Article 1</a></li>
            <li><a href="#">Related Article 2</a></li>
        </ul>
    </aside>
</article>

Footer Element

The footer element represents the footer of a document or section, typically containing copyright information, links to related documents, or contact information.

Example Footer

<footer>
    <p>&copy; 2026 Your Company Name</p>
    <nav>
        <ul>
            <li><a href="privacy.html">Privacy Policy</a></li>
            <li><a href="terms.html">Terms of Service</a></li>
            <li><a href="contact.html">Contact</a></li>
        </ul>
    </nav>
</footer>

Figure and Figcaption

The figure element represents self-contained content (like images, diagrams, code listings) that is referenced from the main content. The figcaption element provides a caption for the figure.

Example Figure with Caption

<figure>
    <img src="diagram.png" alt="Architecture diagram">
    <figcaption>Figure 1: System architecture overview</figcaption>
</figure>

Details and Summary

The details element creates a disclosure widget that the user can open and close. The summary element provides a visible heading for the details.

Complete Semantic Page Structure

Here's how all these elements work together in a typical webpage structure:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Page Title</title>
</head>
<body>
    <header>
        <h1>Site Title</h1>
        <nav>
            <ul>
                <li><a href="#">Home</a></li>
                <li><a href="#">About</a></li>
            </ul>
        </nav>
    </header>
    
    <main>
        <article>
            <h2>Article Title</h2>
            <p>Article content...</p>
            
            <section>
                <h3>Section within article</h3>
                <p>Section content...</p>
            </section>
        </article>
        
        <aside>
            <h3>Related Content</h3>
            <p>Sidebar content...</p>
        </aside>
    </main>
    
    <footer>
        <p>&copy; 2026 Company Name</p>
    </footer>
</body>
</html>
